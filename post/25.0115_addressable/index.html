资源配置

  • 功能介绍：通过 AARules 配置文件规则，进行设置不同的 Addressable 组，解决项目的AB划分规则，减少手动操作；

    • AARules：

      • SingleList：此列表内每个文件都单独进行打AB包；

        • 适合大图

      • LabelList：此列表内根据标签分类进行打AB包；

        • 适合跨文件夹打AB包；

      • FolderList：此列表内根据文件夹分类进行打AB包；

        • 适合按模块划分打AB包

      • 参数说明：

        • Number：第几层文件夹，0则是当前文件夹下查找，1则是第一层文件夹下查找；

        • Search Pattern：搜索规则，搜索什么资源文件；

        • Option：匹配模式，仅当前目录 or 搜索当前目录包括子目录；

          • 1

            Directory.GetFiles(string path, string searchPattern, SearchOption searchOption)

    • Addressable Group：

      • AAGroupSingle：由 SingleList 列表产生的组；
      • AAGroupLabel：由 LabelList 列表产生的组；
      • AAGroupFolder：由 FolderList 列表产生的组；
      • Default Local Group：默认本地组，标记 Addressable 资源中转站组；在 资源标记 栏目内会介绍；（打包实际内用不到）

  • 前期准备：

    • 导入工程 Package 包，并查看 01_ResRules 文件夹；
    • 将导入的 AARules 配置文件覆盖工程内 AARules 配置；
      • AARules 路径：Game -> Settings -> Addressables -> AARules；

  • 打 AA 包：

    • 点击菜单栏 -> Tools -> 资源项 -> 打 AA 包；
      • 等待运行结果；
    • 点击菜单栏 -> Window -> Asset Management -> Addressables -> Groups
    • 即可查看到组内已经打好 AA 包了；

  • 验证结果：

    • 结果：根据 AARules 规则成功打出对应的 AB 包；

    • 查看 AARules 规则配置；

    • 查看 Addressable Group 结果；

    • 查看导出的 AB 包结果；

      • 点击 Build -> New Build -> Default Build Script

using DCFrame;
using UnityEngine;

public class AddressableAssetLoad : MonoBehaviour {
    public async void Start() {
        // 加载贴图
        var address = Asset.GetSpritePath("Addressables/01_ResRules/Folder/Label/animal-skull");
        Sprite obj = await Asset.LoadAsset(address) as Sprite;
        // 加载预制件
        address = Asset.GetPrefabPath("Addressables/01_ResRules/Single/Res/Prefab/Prefab");
        GameObject go = await Asset.LoadAsset(address) as GameObject;
        Instantiate(go, transform);
        // 加载文本
        address = Asset.GetTxtPath("Addressables/01_ResRules/Single/Res/Txt/Txt");
        TextAsset txt = await Asset.LoadAsset(address) as TextAsset;
        Debug.LogWarning(txt.text);
    }
}

资源标记

  • 功能介绍：在 Addressable Group 内标记为文件夹地址的，当在此文件夹内导入的时候，会自动标记为 Addressable，而在 AARules 内 SingleList 内配置文件夹下时，虽然导出是单资源，但是是以文件夹栏目导出的单资源；因此在标记文件夹的单资源时候，继续导入资源无法标记 Addressable 需要手动标记或者点击 打 AA 包才可实现；

    25.02.09 ：新增支持 LabelList 与 FolderList 的文件夹的 number 深度层级在大于1的情况下，在导入的时候也需要标记 Addressable；

  • 解决问题：解决在导入资源的时候，需要手动按需标记 Addressable 的问题；
