前言

前段日子在公司开发一个功能时候，用到了旋转相关的知识，虽然知识面不是很大，不过在开发功能中还是遇到了不少的问题，所以觉得还是整理下以防后续忘记；（这篇算是2022年最后一篇文章了）

Unity API

  • 指定转向到什么角度：gameObject.transform.rotation = Quaternion.Euler(x, y, z)

  • 以当前角度旋转角度：gameObject.transform:Rotate(Vector3.forward * x)

    • Vector3.forward ：Vector3(0, 0, 1)，相当于是旋转Z轴 * x 度；

  • 掉入一个坑：在 Inspector 面板的 Rotation 数值中，360°是一个循环；因此值的范围应该是 0 ~ 360，但是实际运行过程中，发现当运行时候数值超过180度的时候，面板数值则变为负数了；

    • 例如：当实际运行为90°：Inspector 面板显示 90°

    • 例如：当实际运行为300°：Inspector 面板显示 -60°

      • 总结：在超出或等于180°的时候，面板则 显示度数 = 当前度数 - 360
      • 因此需要矫正负数的显示行为，使其得到依旧是 0 ° ~ 360 °

      --- 检视面板旋转角度转换实际角度
function main:getAngleByInspectAngle(angle)
    if angle >= 0 then
        return angle
    end
    return angle + 360
end

--- 实际角度转换检视面板旋转角度
function main:getInspectAngleByAngle(angle)
    if angle < 180 then
        return angle
    end
    return angle - 360
end

DoTween API

  • 在 Doc 的 Transform Rotate 模块中 有个 DORotate 的函数，从截图中可得出，以下为注意点：

    • 如果需要旋转超过360°则 RotateMode 需要选择：RotateMode.FastBeyond360，否则则是默认的RotateMode.Fast，无法超过360度；

    • 如果只是需要旋转很小的度数的话，则需要选择 DORotateQuaternion 代替这个函数；

    • Requires a Vector3 end value, not a Quaternion

      • 掉入另一个坑：这句话中 官方显示了标红，当初开发的时候没有仔细看，导致出现了无法理解的问题；这句话的意思是：需要给定一个 Vector3 类型的结束值；是的，这是一个结束值；当我用UnityAPI Rotate() 函数的时候，这个函数给的是旋转角度值，而 DORotate 是给个结束值，导致我惯性的以为DORotate也是旋转角度值，导致出现了一些误会；
      • 因此假设需要沿着物体当前角度选择180度，则 旋转角度 = 当前物体角度 + 需要旋转的角度；
