Frame 搬运说明

2025-01-14
Unity

文章目录

说明

在 Gitlab 上的 DC、DCFrame 工程，准备在一步一步的搬运到 Github 上了；

  • 搬运的原因如下：
    • 统一路径：由于当前博客是在 Github 仓库上；而 DC 框架在 Gitlab 上。两端提交的长远结果便是不断地换网站、登账号，是有点不方便；
    • 年终汇总：24年年底的时候，在查看两端仓库提交贡献数据时。发现分散的提交贡献在分析其数据的时候不太好处理；
    • 公开仓库：该仓库在 Gitlab 上已经是公开了，出于该仓库不仅限在 DC 博客上做唯一的入口；因此转移到 Github 网站内，想让更多人通过搜索便可发现；
  • Github 工程说明：
    • 工程结构：DC 、DCFrame ，依旧分为两个仓库；目的是：单独拉取 DCFrame 框架便可使用核心内容；若额外拉取 DC 仓库则可作为游戏项目的初期工程；
    • 版本记录：除了在 DC 仓库内的 Version 文件内记录当前仓库的版本内容外，还在 Github 仓库页的 Releases 上记录；
    • 文档说明：对于工程内的一些版本迭代，出一份使用文档。当前文档的目的在于让使用者能更快的入手当前框架；
