    		/// <summary>
        /// 加载 资源 本地化
        /// </summary>
        public static async UniTask<T> LoadAsset<T>(string key) where T : Object {
            if (!AssetTableNameDic.TryGetValue(typeof(T), out var tableName)) {
                Debug.LogError($"未为类型 {typeof(T)} 配置本地化表名");
                return null;
            }
            var handle = LocalizationSettings.AssetDatabase.GetLocalizedAssetAsync<T>(tableName, key);
            await handle.Task;
            if (handle.Status == AsyncOperationStatus.Succeeded) {
                return handle.Result;
            }
            Debug.LogError($"加载本地化资源失败: {key} ({typeof(T)})");
            return null;
        }

Image 本地化

下面以 Image 资源为例介绍本地化流程，其他资源类型的处理步骤基本一致。

项目工程：链接

创建资源

  1. 找到 Game\Localize 文件夹，并在其文件夹下创建名为 Sprite 的文件夹；
  2. 在 Sprite 文件夹下，创建 Zh-CN （存放中文资源）和 En （存放英文资源）
    • 找到 Game\Settings\UIManager\Icon 下的2个贴图资源文件；PrefabRoot 和 PrefabVariable；
    • 以下以这两个资源作为本地化示例。
  3. 把 PrefabRoot 复制粘贴到 Zh-CN底下，把 PrefabVariable 复制粘贴到 En 底下；更改 PrefabVariable 重命名为 PrefabRoot；并在 Inspector 面板中为这两个资源勾选 Addressable。

导表工具

  • 菜单栏：Tools -> 资源项 -> 本地化资源生成：
  • 规则工具：LocalizeRules -> 本地化资源生成：
  • 本地化资源生成规则：
    • 遍历 Game\Localize 目录下除 Text 以外的所有文件夹；
      • 假如遍历到 Sprite 文件夹，则在当前 Sprite 文件夹内的 Table 文件夹下创建名为 Sprite 和 Sprite Shared Data 的 localization 资源数据；
        • 在 Sprite 文件夹下，如果存在本地化语言文件夹名；则在当前文件夹下的 Table 文件夹下创建当前本地化资源对应语言的 localization 资源数据，如：Sprite_zh-CN；
        • 遍历本地化语言文件夹下的资源，将每个资源名称作为 Key，并将资源作为对应语言的数据存入相应的本地化资源表中。
        • 具体事例如下介绍。

生成本地化

  1. 根据导表工具这一栏，点击本地化资源生成；

  2. 可见在 Game\Localize\Sprite\Table 文件夹下，生成了对应的本地化资源；

  3. 打开 Game\Localize\Sprite\Table\Sprite.asset 可见绑定了相关的本地化资源数据；

    • Sprite_en：存放了英文的资源数据
    • Sprite_zh-CN：存放了中文的资源数据

    public class MainGame : MonoSingleton<MainGame> {
    private async UniTask Start() {
        await TestLocalize();
    }

    private async UniTask TestLocalize() {
        var sprite = await Localize.LoadAsset<Sprite>("PrefabRoot");
        if (sprite) {
            Debug.Log(sprite.name);
        }
    }
}

  2. 运行游戏后查看输出日志：PrefabRoot 说明代码已经成功加载对应的本地化资源。

预制件调用

  1. 在场景中创建一个带有 Image 组件的游戏对象。

  2. 点击组件右上角的三个点，选择 Localize 或 AddLocalize。

  3. 选择 Localize，系统会自动添加 Localize Sprite Event 组件，并绑定 Image.sprite 赋值函数；

  4. 选择AddLocalize，该工具除了执行 Localize 操作外，还会查找Sprite 本地化资源内，与 Image 的资源名一致的Key，并自动赋值到 TableEntryReference 中。

  5. 运行游戏并切换语言后，即可看到不同语言对应的本地化资源被加载。
