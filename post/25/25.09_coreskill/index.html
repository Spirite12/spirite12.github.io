文档记录

  • 这篇文章记录 Codex Skill 在当前 Unity 工程里的落地方式；
  • 本文重点是 Core Skill 这一层基础 Skill，以及如何把 Table 和 Localization 的接入流程串起来；

Codex Skill 简介

  • Codex Skill 可以理解为“给 AI 准备的工程使用说明书 + 执行入口”。
  • 它不直接替代项目代码，而是把下面这些内容沉淀成稳定约束：
    • 工程目录应该怎么读；
    • 哪些目录可以改，哪些目录只读；
    • 某类需求应该从哪个模块入口进入；
    • 哪些自动化脚本可以直接执行。
  • 这样做的目的，是让 AI 在多轮协作里不需要每次都重新摸索工程结构，也能减少越界改动和重复分析。

Core Skill 的作用

  • Core Skill 是当前工程里最基础的一层 Skill。
  • 它的目标不是解决单一功能，而是先帮助 AI 建立“进入工程之前的最小共识”。
  • 在功能开发的时候，AI 可以读取此 Skill 然后进行一系列的游戏或者框架开发；

目录划分

  • .agents/skills/core/SKILL.md
    • 入口规范，定义使用顺序、硬性约束、业务约束和完成定义。
  • .agents/skills/core/references/project-map.md
    • 工程总目录和读写边界。
  • .agents/skills/core/references/framework.md
    • 框架层模块入口。
  • .agents/skills/core/references/table.md
    • 表系统、导表规则、生成代码和查表接入顺序。
  • .agents/skills/core/references/localization.md
    • 文本本地化、资源本地化、目录规则和执行顺序。
  • .agents/skills/core/references/ui.md
    • UI 与 各个模块的联动方式。
  • .agents/skills/core/scripts/run_unity_task.py
    • Unity 命令行自动化入口，支持 table、localize、init-localize 等任务。
  • .agents/skills/core/agents/openai.yaml
    • 提供给 Skill 体系的辅助配置入口。

这样划分的目的

  • SKILL.md 负责定规则，不塞过多实现细节。
  • references 负责按模块拆知识，避免一个大文档越写越杂。
  • scripts 负责把可重复执行的 Unity Editor 操作脚本化，减少口头步骤和人工遗漏。
  • 渐进式的披露，使得AI 处理新需求时可以先定位模块，再跳到对应说明，不需要一次性把所有工程细节都读完，减少token的消耗；
