功能构思

  • 在客户端的表逻辑里；表逻辑 = 表数据 + 表对应封装函数；
  • 表数据 ：存放在 CSV 内，每个表内容 = 表字段 + 字段数据；
    • 由策划配表，客户端通过 CsvHelper 读取表内数据；
  • 表函数 ：客户端需要对不同的表定义对应的类，并封装相关通用函数；
    • 出于不同表封装的函数大体一致的情况下，想出来将表类的通用函数通过配置文件生成，而表特有的函数，则是由开发人员单独写；这样减少开发人员写通用函数的时间；

工程介绍

  • Assets
    • DCFrame
      • Editor -> Modules -> Table
        • TableEditor：分析配表规则数据，并生成表对应的类通用函数代码；
        • TableRulesEditor：用于加载UI显示类，并记录表规则数据；
        • TableRulesTypeXXX：XXX对应的是表规则类型对应的写入和UI显示代码；
        • ITableType：接口类，定义TableRulesTypeXXX相关所需的函数；
      • Modules -> Table
        • ScientificToLongConvert：对科学计数法的处理；
        • TableBase：表基类；
        • TableBaseSingle：表单例基类；
        • TableRules：表规则数据定义；
      • Utility
        • TableUtil：表常量和工具类；
    • Game
      • Table : 存放配表数据的文件夹；
      • Settings -> Table
        • TableRules : 配表规则通用数据存放；
      • Scripts
        • Table ：生成的表通用函数存在的文件夹；

菜单栏工具

  • 菜单栏 -> 导表按钮：读取表配置TableRules数据，并依次生成表类代码；

通用逻辑

  • 提示说明：介绍工具内部分按钮的效果；

  • 查、增、删表：对输入字符串的表进行一系列的操作；

  • 删除无用表：遍历表列表内的表规则数据，查找是否有对应的配表，无则删除；

  • 删除表配置：删除当前选择的表配置；

  • 打开表代码、CSV：打开当前选择的表代码或者表 CSV;

  • 保存表数据：将当前更改的数据保存；

  • 刷新表数据：重新进行读取CSV代码的操作；

    • 例：当开发人员开着此窗口，并对CSV进行更改时候，需要点击刷新表配置；

  • 一键导表：与上方菜单栏的导表按钮功能一致；

  • 表列表：列举出当前所有的表规则数据名；

  • 表类型：目前支持到处：默认表、常量表、枚举表；介绍如下；

using System.Collections.Generic;

             return keyDicIsTure2[(isTure, id)];

问题记录

  • Csv 表格文件内的科学计数法问题整合：

    • 问题：当你把数字（特别是较大的数或带较多小数的浮点数）写入 .csv 文件后，用 Excel 打开时，就会变成科学计数法，导致解析的时候会缺少一些精确度；

      原本写入的值 Excel 显示结果
      1234567890123 1.23457E+12
      0.0000000123 1.23E-08

    • 原因：CSV 文件本身是纯文本，Excel 打开 CSV 时，会自动根据内容推断类型，如果数字太大，就显示成科学计数法；

    • 解决：

      1. 在 Csv 内需要显示科学计数法的数字前面加标识符，如： ' (TableUtil.ScientificSign)前缀来避免科学计数，以至于识别成字符串；
        • 项目内目前使用的是这个办法解决；
      2. 先打开 Excel，在通过 Excel 打开此 Csv 文件，然后打开的时候会出现向导，在向导内填写当前字段对应的类型是文本即可；
