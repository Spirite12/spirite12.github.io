功能构思

  • 在客户端的表逻辑里；表逻辑 = 表数据 + 表对应封装函数；
  • 表数据 ：存放在 CSV 内，每个表内容 = 表字段 + 字段数据；
    • 由策划配表，客户端通过 CsvHelper 读取表内数据；
  • 表函数 ：客户端需要对不同的表定义对应的类，并封装相关通用函数；
    • 出于不同表封装的函数大体一致的情况下，想出将表类的通用函数通过配置文件生成，而表特有的函数，则是由开发人员单独写；这样减少开发人员写通用函数的时间；

工程介绍

  • Assets
    • DCFrame
      • Editor -> Modules -> Table
        • TableEditor：分析配表规则数据，并生成表对应的类通用函数代码；
        • TableRulesEditor：用于显示配表规则UI显示，并记录表规则数据；
      • Modules -> Table
        • ScientificToLongConvert：对科学计数法的处理；
        • TableBase：表基类；
        • TableBaseSingle：表单利基类；
        • TableConst：表常量集齐通用函数；
        • TableRules：表规则数据定义；
    • Game
      • Table : 存放配表数据的文件夹；
      • Settings -> Table
        • TableRules : 配表规则通用数据存放；
      • Scripts
        • Table ：生成的表通用函数存在的文件夹；

工具介绍

  • 菜单栏 -> 导表按钮：读取表配置数据，并依次生成表类代码；

  • 表规则工具：TableRules

    • 删除无用表配置 ：遍历表列表内的表规则数据，查找是否有对应的配表，无则删除；
    • 数据类型：代码会根据配表自动识别当前字段的数据类型，也可手动更改；
    • 本地化：勾选则是说明当前字段文本需要识别成多语言进行处理；
    • 主Key：一个表内有且仅有一个主要的Key，用来作为Key存储表数据；
    • 副Key：
      • None：没有副Key；
      • Vice：有多Key，标记副Key的索引值，只生成一个最终的多Key函数；
      • ViceWithList：有多Key，根据索引值生成从单个到最终多Key的多个函数；
    • 获取最大值：
      • None : 不需要获取最大值
      • Single ：当前字段需要获取表数据内的最大值；（提前获取最大值变量；这样就不需要每次读取函数遍历了）
    • 表列表：列举出当前所有的表规则数据名；
    • 保存表数据：将当前更改的数据保存；
    • 一键导表：与上方导表按钮功能一致

问题记录

  • Csv 表格文件内的科学计数法问题整合：

    • 问题：当你把数字（特别是较大的数或带较多小数的浮点数）写入 .csv 文件后，用 Excel 打开时，就会变成科学计数法，导致解析的时候会缺少一些精确度；

      原本写入的值 Excel 显示结果
      1234567890123 1.23457E+12
      0.0000000123 1.23E-08

    • 原因：CSV 文件本身是纯文本，Excel 打开 CSV 时，会自动根据内容推断类型，如果数字太大，就显示成科学计数法；

    • 解决：

      1. 在 Csv 内需要显示科学计数法的数字前面加标识符，如： ' 前缀来避免科学计数，以至于识别成字符串；
      2. 先打开 Excel，在通过 Excel 打开此 Csv 文件，然后打开的时候会出现向导，在向导内填写当前字段对应的类型是文本即可；
