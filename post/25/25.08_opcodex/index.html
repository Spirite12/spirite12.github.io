说明

这次借助 Codex 对当前工程做了一轮整理与修复；

  • Codex 是什么：
    • 它可以直接读工程代码、定位引用关系、批量修改脚本、补测试，再把结果整理成可追踪的变更。
    • 这次我用它来分析 DCFrame 和 Game/Scripts 里的自研脚本，第三方插件代码没有动。
  • 本次分析消耗：
    • 总计消耗了大约 10% 的额度；
    • 使用的模型是 GPT-5.3-Codex。
  • 这次对工程做的整体优化：
    • 统一了自定义枚举命名，去掉了 Enum 前缀，使得后续代码读起来更直观。
    • 基于 Version.md 里记录过的功能模块，补充了测试用例。
    • 修了一批容易引发后续问题的实现点，例如：事件派发、缓存初始化、文本过滤、路径拼接和红点 key 解析。
    • 顺手把一些边界条件补齐，让这些模块在异常输入下更稳一点。

枚举命名

由于 C# 里 Enum 本身已经是一个明确的类型概念，再在枚举名里重复 Enum 会显得有些冗余，因此本次把工程内自研脚本里所有 EnumXXXX 风格的枚举统一去前缀，改成更短的命名方式；

  • 调整目标：
    • EnumTableType -> TableType
    • EnumKeyType -> KeyType
    • EnumFieldType -> FieldType
    • EnumSaveType -> SaveType
    • EnumPrefixPath -> PrefixPath
    • EnumLocaleCode -> LocaleCode
    • EnumPopEnum -> PopEnum
  • 效果：
    • 枚举名更符合当前代码风格；
    • 配合编辑器工具和表规则代码时，阅读成本更低；
    • 相关引用也一起更新，避免“类型名改了但调用没改”的断层。

    public abstract class VersionedTestBase {
        [SetUp]
        public void BaseSetUp() {
            EventMgr.Clear();

        private static void ResetTextFilterRoot() {
            FieldInfo rootField = typeof(TextFilter).GetField("Root", BindingFlags.NonPublic | BindingFlags.Static);
            TextTrieNode root = (TextTrieNode)rootField.GetValue(null);
            root.Children.Clear();
            root.IsWord = false;
        }
    }

Bug 修复

Codex 根据这些模块的实际表现，修复了一系列潜在问题；

EventMgr

原来的实现里，事件派发时如果回调内部又增删监听，容易碰到遍历期间修改集合的问题。

  • 修复前：

    • 事件派发时直接遍历原始监听列表；

    • 回调里如果移除监听，容易影响当前遍历过程；

          /// <summary>
    /// 事件触发（不需要参数的）
    /// </summary>
    public static void DispatchEvent(string eventName) {
        List<Delegate> callbacks = SnapshotCallbacks(eventName);
        if (callbacks == null) {
            return;
        }
        try {
            for (int i = 0; i < callbacks.Count; i++) {
                Delegate callback = callbacks[i];
                (callback as Action)?.Invoke();
            }
        } finally {
            ReleaseSnapshotList(callbacks);
        }
    }

    private static List<Delegate> SnapshotCallbacks(string eventName) {
        if (!eventDic.TryGetValue(eventName, out List<Delegate> value) || value.Count == 0) {
            return null;
        }
        List<Delegate> tempList = GetSnapshotList();
        tempList.AddRange(value);
        return tempList;
    }

    private static List<Delegate> GetSnapshotList() {
        if (dispatchSnapshotPool.Count > 0) {
            List<Delegate> list = dispatchSnapshotPool.Pop();
            list.Clear();
            return list;
        }
        return new List<Delegate>(SnapshotInitCapacity);
    }

    private static void ReleaseSnapshotList(List<Delegate> list) {
        if (list.Capacity > MaxSnapshotCapacity) {
            list = new List<Delegate>(SnapshotInitCapacity);
        } else {
            list.Clear();
        }

        dispatchSnapshotPool.Push(list);
    }

    private static readonly Dictionary<string, List<Delegate>> eventDic = new();
    /// <summary>
    /// 快照池
    /// </summary>
    private static readonly Stack<List<Delegate>> dispatchSnapshotPool = new();
    /// <summary>
    /// 快照初始容量
    /// </summary>
    private const int SnapshotInitCapacity = 64;
    /// <summary>
    /// 最大快照容量
    /// </summary>
    private const int MaxSnapshotCapacity = 256;

  • 思路总结：

    • 核心目标是“安全派发 + 尽量低 GC”。
    • 这个方案比一开始的方案和每次 ToArray() 都更适合高频事件。
    • 后续如果事件量继续增长，也方便继续扩展成更完整的对象池策略。

RedTip

红点 key 的解析方式也补强了一下。

  • 修复前：

    • 通过简单分割解析 key；
    • 当红点名称里带下划线时，容易解析错。

  • 修复后：

    • 改为从最后一个下划线处分割；
    • 名称里包含下划线也能正确解析。

FileUtil 和 Asset

这两个是工程里很常见的基础工具，Codex 顺手把路径行为统一了一下。

  • FileUtil.ReadFile：
    • 只有在不是全路径时才会再次拼接路径；
    • 避免全路径读取时被二次拼接。
  • Asset.GetPath：
    • 路径统一转成 /；
    • 更适合 Unity 里资源地址的习惯写法。

优化处理

TextFilter

  • 屏蔽词库从单文件结构，拆成了“全局词库 + 地区词库”的方式：
    • TextFilter_global.txt
    • TextFilter_zh-CN.txt
  • 这样拆的目的，是把所有地区通用的词和某个地区特有的词分开维护。
  • 运行时初始化逻辑也做了调整：
    • 固定先加载全局词库；
    • 再根据 LocalizationSettings.SelectedLocale 或默认地区加载当前语言词库；
    • 重复初始化前会先清空 Trie，避免旧数据残留；
    • 空行和以 # 开头的注释行会自动跳过。
  • 对应下来，这次 TextFilter 的改动不只是换文件名，而是把它正式接入到了本地化地区的上下文里。

Addressable

  • Asset.LoadAsset 从原来的弱类型加载，改成了泛型 LoadAsset<T>。

    • 这样调用方不需要再手动做类型转换；
    • MainGame、UIBase 这些地方也能直接拿到目标类型。

  • Addressable 资源管理新增了两层处理：

    • 加载中去重：如果同一地址正在加载，会复用已有加载句柄，不会重复发起请求；
    • 引用计数 + 延迟释放：资源不再用完立刻释放，而是记录引用次数，在 FixedUpdate() 里做延迟回收。

  • 路径辅助函数也做了统一：

    • EnumPrefixPath 改成了 PrefixPath；
    • 新增统一的 GetPath()；
    • 所有路径最后都规范成 /，避免 Windows 反斜杠混入 Addressable 地址。

  • UI 侧也顺手补了一层配合：

    • UIBase 会记录自己加载过的资源地址，并在卸载时调用 Asset.Release()；
    • UIMgr 对缺少 UIOrder 组件的节点加了保护，避免排序阶段直接报错。

    using System.Collections.Generic;
using System.IO;
using Cysharp.Threading.Tasks;
using UnityEngine;
using UnityEngine.AddressableAssets;
using UnityEngine.ResourceManagement.AsyncOperations;
using Object = UnityEngine.Object;

namespace DCFrame {
    public class Asset {
        /// <summary>
        /// 根据地址加载资源
        /// </summary>
        public static async UniTask<T> LoadAsset<T>(string address) where T : Object {
            // 获取是否已经加载到
            if (AssetDic.TryGetValue(address, out var assetRef)) {
                assetRef.count = Mathf.Max(0, assetRef.count) + 1;
                assetRef.releaseTime = -1f;
                return assetRef.handle.Result as T;
            }
            // 获取是否正在加载中
            if (LoadingDic.TryGetValue(address, out var loadingRef)) {
                loadingRef.count = Mathf.Max(0, loadingRef.count) + 1;
                await loadingRef.handle.Task;
                if (loadingRef.handle.Status != AsyncOperationStatus.Succeeded) {
                    Debug.LogError($"加载资源失败，地址是: {address}");
                    return null;
                }
                if (!AssetDic.TryGetValue(address, out var loadedAssetRef)) {
                    GetAssetRef(address, loadingRef.handle, loadingRef.count);
                    LoadingDic.Remove(address);
                    loadedAssetRef = AssetDic[address];
                }
                return loadedAssetRef.handle.Result as T;
            }
            // 加载一个新的
            var handle = Addressables.LoadAssetAsync<T>(address);
            var pendingRef = new LoadingRef {
                handle = handle,
                count = 1
            };
            LoadingDic[address] = pendingRef;
            await handle.Task;
            LoadingDic.Remove(address);
            if (handle.Status != AsyncOperationStatus.Succeeded) {
                Debug.LogError($"加载资源失败，地址是: {address}");
                return null;
            }
            GetAssetRef(address, handle, pendingRef.count);
            return handle.Result;
        }

        /// <summary>
        /// 获取一个实例
        /// </summary>
        private static void GetAssetRef(string address, AsyncOperationHandle handle, int count) {
            AssetDic[address] = new AssetRef {
                handle = handle,
                count = count,
                releaseTime = count > 0 ? -1f : Time.realtimeSinceStartup
            };
        }

        /// <summary>
        /// 根据地址卸载资源
        /// </summary>
        public static void Release(string address) {
            if (!AssetDic.TryGetValue(address, out var assetRef)) {
                Debug.LogWarning($"未找到资源: {address}");
                return;
            }
            assetRef.count--;
            if (assetRef.count > 0) {
                return;
            }
            assetRef.count = 0;
            assetRef.releaseTime = Time.realtimeSinceStartup;
        }

        /// <summary>
        /// 释放计时处理
        /// </summary>
        public static void FixedUpdate() {
            if (AssetDic.Count == 0) {
                return;
            }
            var now = Time.realtimeSinceStartup;
            removeList.Clear();
            foreach (var kv in AssetDic) {
                var assetRef = kv.Value;
                if (assetRef.count != 0 || assetRef.releaseTime < 0f) {
                    continue;
                }
                if (now - assetRef.releaseTime >= ReleaseDelay) {
                    Addressables.Release(assetRef.handle);
                    removeList.Add(kv.Key);
                }
            }
            for (int i = 0; i < removeList.Count; i++) {
                AssetDic.Remove(removeList[i]);
            }
        }

        #region 加载前缀和函数

        /// <summary>
        /// 项目前缀地址
        /// </summary>
        public enum PrefixPath {
            Single = 0,
            GamePrefab = 1,
            Settings = 2,
            ScriptTemplates = 3,
        }

        /// <summary>
        /// 前缀与地址的映射
        /// </summary>
        private static readonly Dictionary<PrefixPath, string> PrefixPathDic = new() {
            { PrefixPath.Single, "Assets/Simple/"},
            { PrefixPath.GamePrefab, "Assets/Game/Prefabs/"},
            { PrefixPath.Settings, "Assets/Game/Settings/"},
            { PrefixPath.ScriptTemplates, "Tools/ScriptTemplates/"}
        };

        /// <summary>
        /// 获取资源加载地址
        /// </summary>
        public static string GetPath(string path, PrefixPath enumPrefix = PrefixPath.GamePrefab) {
            if (!PrefixPathDic.TryGetValue(enumPrefix, out string prefixPath)) {
                Debug.LogError($"暂未支持当前前缀枚举： {enumPrefix}");
                return "";
            }
            return Path.Combine(prefixPath, path).Replace("\\", "/");
        }

        /// <summary>
        /// 获取贴图加载地址
        /// </summary>
        public static string GetSpritePath(string path, PrefixPath enumPrefix = PrefixPath.GamePrefab) {
            return GetPath(path + ".png", enumPrefix);
        }

        /// <summary>
        /// 获取文本加载地址
        /// </summary>
        public static string GetTxtPath(string path, PrefixPath enumPrefix = PrefixPath.GamePrefab) {
            return GetPath(path + ".txt", enumPrefix);
        }

        /// <summary>
        /// 获取预制件加载地址
        /// </summary>
        public static string GetPrefabPath(string path, PrefixPath enumPrefix = PrefixPath.GamePrefab) {
            return GetPath(path + ".prefab", enumPrefix);
        }

        /// <summary>
        /// 获取资源加载地址
        /// </summary>
        public static string GetAssetPath(string path, PrefixPath enumPrefix = PrefixPath.GamePrefab) {
            return GetPath(path + ".asset", enumPrefix);
        }

        #endregion


        /// <summary>
        /// 资源计数
        /// </summary>
        private static readonly Dictionary<string, AssetRef> AssetDic = new();
        private static readonly Dictionary<string, LoadingRef> LoadingDic = new();
        private static readonly List<string> removeList = new();
        /// <summary>
        /// 资源释放时间
        /// </summary>
        private const float ReleaseDelay = 60f;

        /// <summary>
        /// 资源引用类
        /// </summary>
        private class AssetRef {
            /// <summary>
            /// 资源句柄
            /// </summary>
            public AsyncOperationHandle handle;
            /// <summary>
            /// 引用次数
            /// </summary>
            public int count;
            /// <summary>
            /// 释放时间
            /// </summary>
            public float releaseTime;
        }

        /// <summary>
        /// 正在加载中的资源引用
        /// </summary>
        private class LoadingRef {
            public AsyncOperationHandle handle;
            public int count;
        }

    }

中文注释乱码

  1. 这次修改过程中，Codex 曾经把工程里一部分中文注释写坏过，导致出现乱码。

    • 问题表现：

      • 中文注释显示异常。

    • 处理方式：

      • 先把我这轮改动回退；
      • 再用明确的 UTF-8 方式重新写入；
      • 重新确认中文注释已经恢复。

    • 说明：

      • 后续如果再批量改中文脚本，我会继续保持 UTF-8 写入，避免再次出现类似情况。

总结

  • 在需求明确的情况下，比如删除 Enum 前缀，Codex 能够比较准确地完成目标，执行效果也比较稳定。
  • 在需求相对模糊的情况下，比如“找出潜在的 bug 并修复”，Codex 通常会优先从判空、边界条件和异常路径入手，同时也能覆盖一些潜在的隐性问题，例如 EventMgr 这类事件派发场景。
  • 在编写测试用例时，Codex 偶尔会因为没有完全贴合现有代码实现而产生少量冗余测试，但整体来说，大部分测试用例都能够正常通过，并且对核心功能有较好的回归保障。
  • 在权限处理方面，当需要读取超出当前文件夹范围的内容时，系统会主动提示是否授权，这种权限保护机制比较合理，也比较安全。
