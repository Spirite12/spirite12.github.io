前言

  • 在音效领域内，常用的音效插件有：FMOD Studio、Criware、Wwise、Audio Toolkit；
  • 使用 ChatGPT 对这几款音效插件进行了对比方案输出：
特性 / 插件 CRIWARE (AtomCraft) FMOD Studio Wwise Audio Toolkit
适用游戏规模 中大型 中大型 中大型 小型 / 中型
播放方式 事件驱动 事件驱动 事件驱动 AudioSource
音频存储方式 Bank（ACB 文件） Bank（.bank 文件） Bank（.bnk 文件） Unity AudioClip
3D 音效支持 强 强 最强 基础 3D 支持
动态混音 & DSP 高级 强 最强（行业标杆） 基础淡入淡出
音频对象池优化 支持 支持 支持 支持
CPU / 内存优化 低 CPU & 低内存占用 可能需要手动优化 强优化 适中
支持平台 全平台 全平台 全平台 PC & 移动端
学习成本 高 中等 最高 最低
  • 从以上的对比方案来看，出于以下几点考虑，来选取：Audio Toolkit 作为本文的音效插件推荐：
    • 出于项目做独游的想法，那么上手快，费用低，成了首要的选取标准；
    • 其次 Audio Toolkit 的音效存储方式使用的是 Unity AudioClip；
    • 最后 Audio Toolkit 支持基础所需的音效需求；

使用教程

下载

  • Audio Toolkit 提供 免费 和 付费 版本，其中的区别在于免费版不支持：对象池管理类 与 限于12个音频项目；

    • 本文使用免费版本进行下载；

  • 首先打开免费版网址，登陆 Unity 账号后，点击在 Unity 中打开按钮；

  • 项目内出现 Package Manager 窗口，对着 Audio Toolkit Free Version 点击 Import 导入插件；

知识梳理

  1. QA：Global Audio Settings ：全局参数

    • Additional Audio Controller：若勾选，则代表当前 Controller 是作为附加脚本到主脚本上的，此刻是可以拥有多个 Controller；但请确保主 Controller 只能存在一个。
      • 相当于全局主 Controller 只可以有一个，但是针对不同的场景、UI、系统可以分别有附加脚本；而这些附加脚本只存在以这些情况的生命周期里；
    • Persist Scene Loading：若勾选，则切换到下一个场景的时候，此脚本不会被销毁；即存在场景的 DontDestroyOnLoad 下；需确保此脚本在场景内唯一；
    • Unload Audio On Destroy：若勾选，则当当前脚本被销毁的时候，会从内存内卸载所有的 AudioClips；

  2. QA：Category Settings

    • Audio Controller 类 Category Settings 目录配置介绍：

      • 功能介绍：用以音效配置，此配置分为三大层级；

      • 第一层 Category：申明有哪些目录的音效；

        • 如：背景音、音效等等类别；

      • 第二层 Audio Item Settings ：这个目录下申明一系列音效配置；

        • 如：播放模式、循环模式、延迟和音调等等；
        • 此配置不包含具体 AudioClip 音效；

      • 第三层 Audio Sub-Item Settings : 对指定的某个音效配置赋值 AudioClip 源和当前音效配置；

        • 音效配置内可按序设置多个 AudioClip；

    • 播放 Category Settings 内申明的音效配置：

      • 播放的函数：

        1
2
3
4

        // 播放音效，如果有多个同时播放，则一起播放音效；
AudioController.Play("flute");
// 播放背景音，只能存在一个，以最后调用此函数的背景音为主；
AudioController.PlayMusic("flute");

      • 获取 audioID 值，以上图为例，则是 Audio Item Settings -> Name -> flute；

        • audioID需全局唯一；

  3. QA：Playlist Settings

    • Audio Controller 类 Playlist Settings 播放列表介绍：

      • 功能介绍：按序播放音效列表配置，此配置分为二大层级；

      • 第一层：定义有哪些播放列表；

      • 第二层：在某个播放列表下，有哪些音效需要播放；

        • 音效需要在 Category Settings 中定义才可使用；

    • 播放 Playlisy Settings 内申明的音效配置：

      • 播放的函数：

        1

        AudioController.PlayPlaylist("Playlisy1");

      • 获取 PlaylistName 参数值，以上图为例，则是 Playlisy Settings -> Playlist Name；

  4. QA：Channel Settings

    • Audio Controller 类 Channel Settings 通道设置介绍：
      • 功能介绍：处理当前播放的音效衔接下一段音效的配置；
      • 参数介绍：
        • Crossfade-in Time：淡入，播放音频几秒内，音量从0到1；
        • Crossfade-out Time：淡出，音频结束后几秒内，音量从1到0；
        • 针对 Playlist 配置：
          • Loop Playlist：循环播放列表；
          • Shuffle Playlist：随机播放列表；
          • Crossfade Playlist：淡入淡出播放列表；
          • Delay Betw. Playlist Tracks：当前音频延迟几秒播放下个音频；

项目接入

  • 项目工程：链接

  • 导入包到工程 Assets\Plugins 文件夹下；

    • 删除 Demo 文件夹，减少包体；

  • 新增音效预制件；

    • 路径在：Assets\Game\Settings
      • AudionController 脚本与配置文件功能一致，因此放在此目录下；
    • AudioObject ：用于音乐播放的预制件；
    • AudioControllerMain ：音乐管理预制件；
      • 定义：Music（背景音）、Sound（音效）、System（游戏系统内音效）
        • 当然后续可根据实际情况再次增删改；

  • MainGame.cs 新增加载音效管理代码：

    1
2
3
4
5

    private async UniTask Start() {
    string path = Asset.GetPrefabPath("AudioToolkit/AudioControllerMain", Asset.EnumPrefixPath.Settings);
    GameObject prefab = await Asset.LoadAsset(path) as GameObject;
    Instantiate(prefab, transform);
}

  • 实际开发过程中，会进行以下操作：

    • 资源路径：Assets\Game\Res\Audio 文件夹下
      • 可根据 AudioControllerMain 上的第一大类对文件夹进行分类；
    • 配置处理：打开 AudioControllerMain 预制件进行音效相关的配置处理；
    • 代码调用：
      • 背景音乐：AudioController.PlayMusic();
      • 音效音乐：AudioController.Play();
